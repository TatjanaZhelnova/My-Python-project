# My-Python-project
Basic Python: ETL-process  🚀  

Процессы:  

1. Настройка лога работы скрипта с отлавливанием всех ошибок и выводом промежуточных стадий  
   • Лог сохраняется в тектовый файл  
   • Файл именуется в соответствии с текущей датой  
   • Если в папке с логами уже есть другие логи - они удаляются, остаются только логи за последние 3 дня  
   
2. Обращение к API api_url = "https://b2b.itresume.ru/api/statistics" для получения данных,  
   где  client - Skillfactory (регистр важен)  
        client_key - M2MGWS (регистр важен)  
        start - дата-время начала в формате `2023-04-01 12:46:47.860798`` (время на сервере в нулевом часовом поясе)  
        end - аналогично дата окончания    
3. Обработка и подготовка данных к загрузке в базу данных в PostgreSQL
Структура полученных в результате запроса данных:  
  • user_id - строковый айди пользователя  
  • oauth_consumer_key - уникальный токен клиента  
  • lis_result_sourcedid - ссылка на блок, в котором находится задача в ЛМС  
  • lis_outcome_service_url - URL адрес в ЛМС, куда мы шлем оценку  
  • is_correct - была ли попытка верной (null, если это run)  
  • attempt_type - ран или сабмит  
  • created_at - дата и время попытки   
Делаем валидацию полученных строк на ошибки с контролем заполнения важных полей  

4. Загрузка предобработанных данных в локальную базу PostgreSQL  
   • база развернута локально на компьютере  
   • создаем таблицу (api_process)  
   
5. Формирование необходимых запросов для агрегации данных за определенный период времени
   В данном проекте определены три запроса для составления отчета:  
   •1 Количество уникальных пользователей за день  
   •2 Максимальная активность пользователей по дням  
   •3 Общее количество попыток для каждого пользователя  

6. Агрегирование данных для загрузки в Google Sheeets  
    Результаты запросов формируем в формат словаря  
   
7. Загрузка данных в таблицу Google Sheets  
  • создаем таблицу в Google Sheets (Aggregated Data)  
  • cоздаём Service Account в Google Cloud Console    
  • cкачиваем JSON ключ  
  • даём доступ к таблице (как email сервисного аккаунта)  
  • каждый запрос сохраняем на отдельных листах таблицы  

8. Настройка оповещений о результатах процесса по электронной почте Mail.ru (добавлен вариант с отправкой лога)  
  • формируем структуру письма(тема, содержание, отправитель и получатель)  
  • создаем SSL-контекст для защищенного соединения  
  • устанавливаем соединение с SMTP-сервером   
  • авторизация (создается специальный пароль для почты)  
  • отправляем письмо с результатами процесса
